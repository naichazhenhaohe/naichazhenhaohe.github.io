关于面经-2

发表于 2019-09-27 | 分类于 面经
本文字数： 3.1k | 阅读时长 ≈ 6 分钟

总结一下最近的一场面试。

技术一面

面试官是个很可爱的哥哥 (´ 艸`)，好喜欢他，面的很开心，然后我感觉我们俩都是那种，对面笑起来就会被带动起来一起笑的那种性格，所以导致我感觉我们面试过程中超过 50% 的时间都在笑。

其实忘了好多。。因为聊的很开心，就没怎么留意怎么记，然后这两天天天跑重大有点累，就更加忘了。

  1. position。
    static(default)、relative、absolute、sticky、fixed。
    我忘了 sticky 是怎么定位来着，而且说漏了 fixed，后面又的问题说到又补充了一下。
  2. 清除浮动。
    clean 嘛，然后其实说不上来了 hhh 就说了一个创建 BFC
  3. BFC
    • html 标签
    • float
    • position：absolute/fixed (就是这里说到了 fixed，然后就补充了一下说第一个问题还有一个 fixed，再聊了聊 fixed)
    • overflow 不为 hidden
    • display: inline-box / flex / gird / table-cell / table-caption (这个当时忘了没说。
  4. 盒模型 box-sizing 以及两个属性值的区别
  5. 垂直居中
    • flex
    • grid
    • absolute & transform(-50%,-50%) & top $ left
    • 负 margin (需要宽高已知)
  6. em / rem / px
  7. 原始类型和拓展类型
    有点意思的是 Object 算不算原始类型，《ES6 标准入门》第三版第 183 页，是归了的，所以我也说了，但是哥哥那边的认知是不属于的，但是没有过多的争论 hhh
  8. 闭包
  9. this
  10. 原型
  11. let / const / var
  12. promise
    • 三个状态
    • .then .catch .finally .all .race
  13. 箭头函数
    • 缩略写法
    • 箭头函数的 this 的问题
  14. Array 的方法
  15. Array 的 ES6 新属性、方法。(上一次面试问了来着，但是没好好记，就说了 Array.of & Array.from。
  16. 然后问了还有那些知道了 ES6 的东西
    • Symbol
    • Set
    • Map
    • Generator
    • class
    • async / await
  17. 事件冒泡
  18. 事件循环机制，其实就是宏任务微任务这些，process.nextTick 会是微任务，然后做了这个题，hhh 对的不多，因为当时不知道 process.nextTick 是宏任务还是微任务。
  1. localstorage sessionstorage cookie
  2. React 生命周期
    • 倒是只说了 16.4 之后的生命周期
    • 顺便吐槽了下 static getDerivedStateFromProps 以及 getSnapshotBeforeUpdate 没啥用。。。 (呜呜呜，如果不觉得话，确实是我菜，真的没有遇到过需要的时候
  3. 组件之间通信
    • 父子 => props
    • 子父 => props 回调
    • 祖孙 => context 或者一层一层的 props
    • 兄弟 => 在父组件里处理，或者 context
    • 无关系组件 => redux / 自定义订阅事件(这个当时忘了，没说，而且其实也没实际操作过
  4. Redux 原理
    • 就是 store / action / reducer 三个点组成一个圈的问题
  5. 有什么要问他的
    • 问了问具体在做什么
    • 问了问公司的技术栈(因为我是 ReactJs 的用户以及喜爱者，但是宣讲的时候只提到了 VueJs，面试官哥哥说我 ReactJs 都这么熟悉了学习 VueJs 会很快的
    • 工作地点

其实好像不止这么点，但是整体还是比较顺畅的，有想到的话之后再来补。

HR 二面

面试官是个很有气质的姐姐。
除了些常规问题外，主要是聊聊天。

  1. 上来就问了考虑城市的问题，就是选择 offer 的优先级。
    回答的是优先级浙江最高（因为家在浙江），但是没有收到杭州的 offer，所以去哪里都可以，工作环境和薪资平行考虑。
  2. 家庭情况、父母、兄弟姐妹、是否单身、有没有拿到 offer。
  3. 然后就开始聊天了。。。
    虽然说是聊天，其实是 HR 姐姐在说为主，聊了聊 HR 姐姐在重庆这一周的感觉，说了说她对重庆和北京的差异的看法，聊了聊饮食、租房，其实有感觉到姐姐在向我灌输一些北京的生活压力主要来源于购房的问题，然后也花了一些时间跟我说了说北京，确实有改变一些我对北京的看法吧。
  4. 最后姐姐让我空出第二天早上的时间等待谈 offer

所以其实在面试中后期我就感觉有点怪怪的，或者自大一点说我其实面完就觉得自己已经过了。
因为问了基础情况之后就开始聊天了，HR 姐姐也没有进行一些测试性的问题，也没有针对之前的技术面进行回顾什么的，然后后面的聊天也有在像我介绍一些北京生活压力其实和重庆差不多的感觉，然后对比了下租房、饮食、交通的费用(嘿嘿，为什么聊这个其实我能感觉到啦
而且最后是直接让我空出隔天的早上时间，而我的校友是说的晚上回去等通知。
所以其实这一部分没有什么好总结的，主要还是感谢技术面的那个哥哥给了我很高的评价，另外就是比较凑巧 HR 姐姐他们行程比较赶所以没有聊得很多。

所以也是努力+运气的结果吧。
说起来好久没有这种感觉了，今年一直觉得浑浑噩噩的。

总结

说起来也就是大前天的技术一面到昨天拿到 offer 也就短短的三天，有点梦幻的感觉。
那么其实我的秋招。。。其实就去面了两家公司(虽然投了很多！可恶)就结束了
其实很感谢的是一个同学，那天她带上了我跑到重大参与了宣讲，然后又现场做了笔试，之后就一帆风顺的最后拿到了 offer。
也就是说网申虽然很重要啦，但是跑宣讲其实也很重要的，尤其是去附近的好的学校的校招宣讲，有些还是很值得去的。
当然啦也不能放弃学习啊，这个月真的压力很大，韩语也停掉好久没背单词了，晚上又失眠，前一阵子还连续三天流鼻血，但是还是保持多看岗位信息(信息量很重要！)以及每天都刷刷面经写写博客。
还就是运气，多试试总有运气好的时候。
